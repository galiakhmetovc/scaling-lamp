# Agent OS Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add global agent profiles, agent-bound session creation, agent-owned prompt files and local skills, exact tool allowlists, and canonical inter-agent messaging with loop protection, without introducing a second runtime path.

**Architecture:** Extend the existing modular monolith through the current persistence schema, bootstrap/app layer, prompt assembly, tool catalog/filtering, background-job/inbox wake-up substrate, and TUI/CLI/daemon HTTP surface. Store agent identity as durable global state plus immutable `session.agent_profile_id`, source `SYSTEM.md` and `AGENTS.md` from `agent_home`, and implement `message_agent` as a model-facing tool that routes through the existing background/delegation substrate instead of nesting a second live chat loop.

**Tech Stack:** Rust, rusqlite, serde/serde_json, existing SQLite/file-backed persistence, current daemon HTTP transport, existing background job and session inbox substrate.
